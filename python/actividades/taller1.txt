Taller #1

Construya un API con 5 endpoints.
    1. Que permita saber si el API está activo.
    2. Que permita obtener todas las ciudades de la base de datos.
    3. Que permita obtener el pais de una ciudad que se le pase, y si la ciudad no existe, debe devolver "La ciudad no existe".
    3. Que permita obtener el pais y la ciudad cuando se pase un id, y si el id no existe, debe devolver "El id no existe".
    4. Que permita agregar una nueva ciudad y pais con un Request Parameter.
    5. Que permita eliminar una ciudad y pais, pasando su id con un Query Parameter

Condiciones:
    1. Cada endpoint debe estar documentado.
    2. Cada endpoint debe especificar cuales parametros son obligatorios o no.
    3. Debe haber control de exepciones en el código.
    4. Si ocurre un error, debe arrojar un error 400, con el error que ocurrió.
    5. La busqueda debe funcionar aunque las ciudades que se pasen esten en mayuscula, minuscula completa o solo iniciales.
    6. Si agrego una nueva ciudad y pais, el id, debe ser el número siguiente al ultimo id, no pueden haber ids repetidos.
    7. Si elimino un pais y ciudad, se deben reorganizar todos los ids de las bases de datos.

Extras: 
    1. Puedes cambiar el formato de archivo .txt en la base de datos, al que quieras (.csv, .json, .xlsx, .parquet, .avro), pero siempre usando un archivo.
